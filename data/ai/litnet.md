# Litnet Context

Litnet is a fiction platform for readers and authors. For Digital Shelf, treat Litnet as a place where the user usually wants helper tools for the author-side workflow.

## Core areas that matter for requests

- author profile and author page presentation;
- book cards and book pages;
- annotations, titles, covers, genres, and tags;
- chapter and draft related work;
- author blog posts and communication with readers;
- comments, subscriptions, update tracking, and engagement-related monitoring;
- promotion-oriented actions and materials mentioned in official Litnet announcements and author blog materials.

## Publicly visible author-side signals and patterns

- author pages can contain profile presentation, "about me" style information, and links;
- author communication with readers matters, including blog posts, announcements, and short notices;
- readers can follow authors and monitor updates;
- books are strongly tied to discoverability through metadata, covers, genres, tags, and visibility;
- public Litnet materials repeatedly emphasize promotion, engagement, and discoverability.

## Typical repetitive work an author may want to automate

- filling metadata for books and related publishing forms;
- preparing and reusing sets of tags, descriptions, or promo fragments;
- switching quickly between several books, drafts, stats, and feedback sections;
- checking updates, comments, reader reactions, or publication statuses;
- collecting several routine actions into one helper window or one extension panel.

## Good wording patterns for AI-generated TЗ

When the user writes about Litnet, the AI should often translate the request into developer language such as:

- helper for faster work in the author cabinet;
- tool for repeated actions on book cards or chapter pages;
- extension for quicker navigation between books, drafts, comments, and statistics;
- assistant for preparing tags, annotations, covers, and publication-related fields;
- utility for monitoring reader responses, comments, and update activity.

## Frequent user intentions the AI should recognize

- "мне нужно быстрее оформлять книги" -> metadata helper, templates, mass text preparation;
- "нужно видеть всё в одном месте" -> dashboard / summary panel / quick navigation tool;
- "хочу не забывать что обновилось" -> monitor / tracker / notifications helper;
- "хочу меньше ручной рутины" -> autofill, repeated actions helper, snippet reuse, preset builder;
- "нужна программа для продвижения" -> promo-material prep, blog support, visibility support, tracking engagement.

## Good scope examples for Litnet requests

- helper that groups often-used author pages in one panel;
- tool that prepares repeated annotation or tag sets for books;
- extension that speeds up moving between drafts, books, and reader feedback;
- assistant that helps track comments, updates, or reader reactions;
- utility that simplifies preparing promo posts or support text for author communication.

## Bad scope patterns the AI should avoid

- claiming access to private moderation tools;
- promising automatic actions that require undocumented internal permissions;
- pretending to know exact hidden fields or button names if the user did not mention them;
- turning a simple helper request into a huge CRM-like system unless the user explicitly asked for it.

## Desired output behavior for Litnet

When producing ТЗ for Litnet:

- connect the request to book management, author workflow, navigation, reader communication, or promotion;
- translate emotional phrases into concrete actions;
- preserve the selected Litnet context in wording;
- prefer practical automation and interface helpers over speculative platform integrations.

## Example transformations

Raw user idea:
"Хочу чтобы было удобнее работать с книгами и не лазить везде"

Better AI direction:
"Что нужно сделать:
Сделать инструмент для Litnet, который собирает в одном окне быстрый доступ к основным действиям автора при работе с книгами.

Желательно:
Добавить быстрые переходы между книгами, черновиками, комментариями и статистикой, чтобы уменьшить количество ручных переходов по кабинету."

Raw user idea:
"Нужна штука для тегов и описаний"

Better AI direction:
"Что нужно сделать:
Сделать helper для Litnet, который помогает быстрее заполнять теги, аннотации и повторяющиеся текстовые блоки в карточках книг.

Желательно:
Добавить возможность хранить заготовки часто используемых описаний и быстро подставлять их при оформлении книг."

## Constraints

- Do not claim exact hidden admin controls unless the user described them.
- Prefer neutral wording such as "в кабинете автора", "на странице книги", "в форме редактирования книги", "в блоге автора".
- If the user asks for convenience, assume they want less manual repetition, faster switching, fewer clicks, and clearer visibility of information.

## Useful source direction

This context is based on Litnet public pages and official blog materials describing author pages, following authors, author opportunities, and promotion-related changes.
